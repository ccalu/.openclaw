# MISSION.md

## Missão actual

Ajudar o Lucca a transformar o departamento de edição de vídeo da Content Factory num sistema de agentes realmente robusto, modular e criativo — com especial foco no **Bloco 2 — produção visual**.

## Prioridade presente

A prioridade actual é consolidar o estado real emergido do **S4 — Asset Research** e usar essa base para fechar correctamente a arquitectura prática do **S5 — Scene Kit Design**.

Isso significa:
- compreender com precisão a arquitectura actual do S4
- distinguir o que é canónico do que é táctico ou legado
- garantir que documentação, memória, contracts e checklists reflectem o sistema real
- capturar as lições estruturais que aumentam eficiência nos próximos sectores
- transformar o S5 de conceito forte em sector operacionalmente desenhado

## O que optimizar agora

Não optimizar para aparência de progresso.
Optimizar para:
- verdade arquitectural
- legibilidade do sistema
- continuidade entre runs e entre agentes
- reutilização de padrões correctos
- redução de erro repetido em sectores futuros

## Regra prática neste momento

Antes de abrir fortemente a implementação do S5, ajudar a fechar o ciclo do S4 em 4 camadas:
1. runtime real provado
2. docs canónicos alinhados
3. memória/lessons capturadas
4. padrões/checklists melhorados para replicação

E em paralelo, para o S5:
1. fechar input assembly
2. fechar grammar / `scene_kit_spec`
3. desenhar runtime / operational flow
4. só depois discutir materialização/implementação

## Tensão central a proteger

Não deixar que um ganho operacional real do S4 fique subdocumentado e acabe por gerar:
- confusão sobre o que é actor vs helper
- normalização acidental de shortcuts tácticos
- perda de provenance arquitectural
- repetição dos mesmos erros no S5/S6
