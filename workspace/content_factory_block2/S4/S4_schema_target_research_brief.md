# S4 Schema — Target Research Brief

_Status: planning draft_
_Last updated: 2026-04-04_
_Contract id: `s4.target_research_brief.v1`_

---

## 1. Purpose

This schema defines the canonical target-level brief generated by `op_s4_web_investigator` and consumed by `op_s4_target_research_worker`.

It is the main artifact that turns:
- intake-level target definition

into:
- bounded target-focused discovery work

This contract is critical because it is where the discovery layer stops being abstract and becomes operational.

---

## 2. Producer and consumers

### Producer
- `op_s4_web_investigator`

### Primary consumers
- `op_s4_target_research_worker`
- `sm_s4_asset_research`

### Secondary consumers
- `op_s4_candidate_evaluator`
- debugging/review flows

---

---

## 4. Field specification

## 4.1 `contract_version`
- required
- type: string
- fixed value: `s4.target_research_brief.v1`

## 4.2 `job_id`
- required
- type: string

## 4.3 `batch_id`
- required
- type: string
- identifies which discovery batch this target belongs to

## 4.4 `target_id`
- required
- type: string
- must match a `research_targets[*].target_id` from the intake

## 4.5 `canonical_label`
- required
- type: string

## 4.6 `target_type`
- required
- type: string
- expected to be copied from the intake target definition

## 4.7 `scene_ids`
- required
- type: array of strings
- may be empty only with explicit warning

## 4.8 `research_modes`
- required
- type: array of strings
- copied or refined from intake target definition

## 4.9 `priority`
- required
- type: string
- allowed values:
  - `critical`
  - `high`
  - `medium`
  - `low`

## 4.10 `search_goals`
- required
- type: array of strings
- each item should express one concrete search objective

Examples:
- `find historical exterior views`
- `find interior ballroom wide shots`
- `find newspaper or archival references for closure event`

## 4.11 `research_needs`
- required
- type: array of strings
- restates or sharpens the needs coming from intake

## 4.12 `source_entity_ids`
- required
- type: array of strings
- preserves traceability back to S3 entities

## 4.13 `storage_paths`
- required
- type: object

### Required fields inside `storage_paths`
- `target_root` — absolute path string
- `candidate_set_path` — absolute path string
- `assets_dir` — absolute path string
- `previews_dir` — absolute path string
- `captures_dir` — absolute path string

### Optional fields inside `storage_paths`
- `worker_log_path` — absolute path string
- `worker_status_path` — absolute path string
- `worker_checkpoint_path` — absolute path string

## 4.14 `output_contract`
- required
- type: string
- fixed value: `s4.candidate_set.v1`

## 4.15 `notes`
- required
- type: array of strings
- may be empty

## 4.16 `warnings`
- required
- type: array of strings
- may be empty

---

## 5. Validation rules

1. `contract_version` must match exactly
2. `output_contract` must equal `s4.candidate_set.v1`
3. `target_id` must not be empty
4. `canonical_label` must not be empty
5. `research_modes` must contain at least one value
6. `search_goals` must contain at least one value
7. `storage_paths.target_root` and `storage_paths.candidate_set_path` must be absolute paths
8. all required storage paths must be present even if directories are not yet created by the worker
9. `warnings` and `notes` must always exist

---

## 6. Edge cases

## 6.1 Target with weak scene linkage
If a target exists but cannot be confidently mapped to scenes:
- `scene_ids` may be empty
- `warnings` must explain why
- worker should still be allowed to run if the target remains valuable

## 6.2 Symbolic target ambiguity
For symbolic or abstract targets:
- `search_goals` should be more concrete than the target label
- avoid vague instructions like just `find symbolic references`
- prefer explicit subgoals like:
  - `find empty luxury interiors conveying loss`
  - `find decree/newspaper references for shutdown event`

```json
  "job_id": "job_006_pt_001",

  "research_needs": [

    "iconic facade imagery",
    "large-scale contextual shots"
  ],
  "source_entity_ids": ["l2", "o1"],
  "storage_paths": {
    "target_root": "C:\\...\\targets\\rt_001_hotel_quitandinha",
    "candidate_set_path": "C:\\...\\targets\\rt_001_hotel_quitandinha\\candidate_set.json",
    "assets_dir": "C:\\...\\targets\\rt_001_hotel_quitandinha\\assets",
    "previews_dir": "C:\\...\\targets\\rt_001_hotel_quitandinha\\previews",

  },
  "output_contract": "s4.candidate_set.v1",
  "notes": [
    "Deduped from location/object overlap in S3"
  ],
  "warnings": []
}
```
