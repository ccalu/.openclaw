AGENTS & PROMPTS (5 TOTAL)
================================================================================

All agents receive JSON input, call skills, return JSON output.

1. people_finder
   Input: {person_name, era, context, constraints}
   Skill: archive-image-search
   Output: {candidates[], search_queries[], confidence[]}
   
2. location_finder
   Input: {location, era, context, visual_tone}
   Skill: archive-image-search
   Output: {candidates[], location_features[], confidence[]}
   
3. event_object_finder
   Input: {entity, era, context, query_type (event|object|document)}
   Skill: archive-image-search
   Output: {candidates[], authenticity_notes[], confidence[]}
   
4. video_finder
   Input: {query, era, duration_needed_sec, constraints}
   Skill: video-archive-finder + FFmpeg validation
   Output: {video_candidates[], usable_segments[], ffmpeg_validated}
   
5. asset_judge
   Input: {candidates[] from all 4 finders, constraints}
   Skill: asset-rights-normalizer (verify rights) + heuristics (scoring)
   Output: {final_score, recommendation (USE_REAL|GENERATE_AI), reasoning[]}

Scoring Rubric (asset_judge):
  final_score = (
    relevance × 0.30 +
    accuracy × 0.25 +
    quality × 0.20 +
    rights_safety × 0.15 +
    uniqueness × 0.10
  ) / 10
  
  Decision threshold:
    ≥ 0.72 → USE_REAL
    0.55-0.72 → USE_REAL_WITH_ENHANCEMENT (upscale, color correction)
    < 0.40 → GENERATE_AI

KEY INSIGHTS & RECOMMENDATIONS
================================================================================

1. ARCHIVE-FIRST STRATEGY
   Real images from public archives preferred over AI generation.
   Reason: Authenticity, diversity, legal certainty, zero cost
   Fallback: AI only if real asset scores < 0.72

2. CC0 + PUBLIC DOMAIN PHASE 1
   Avoid CC-BY complexity in early phases.
   Phase 1-2: CC0 only (highest confidence)
   Phase 3+: Can expand to CC-BY with human review layer

3. FIVE SPECIALIZED AGENTS (NOT ONE SUPERAGENT)
   people_finder, location_finder, event_object_finder, video_finder, asset_judge
   Reason: Easier to debug, scale, and improve independently
   Each agent has single responsibility → high quality

4. SCORING THRESHOLD 0.72 (TUNABLE)
   Based on empirical balance: high quality + good coverage
   Will need adjustment after Phase 1-2 real-world testing

5. DEDUPLICATION PHASE 3 (NOT PHASE 1)
   Start simple: URL normalization only
   Add perceptual hashing only if dedup quality insufficient
   Cost: $0 (local computation)

6. COST REDUCTION: 90%+ FOR ASSET DISCOVERY
   Current: $5.55/video (Gemini paid images)
   Future: $0.01/video (archive search + local scoring)
   ROI: Payoff in first 50-100 videos produced

7. DEPENDENCIES ARE CLEAR
   Branch 4 depends on: Branch 3 (visual_plan.json input)
   Branch 4 feeds into: Branch 5 (asset_candidates.json for image generation)
   No circular dependencies; clean separation of concerns
