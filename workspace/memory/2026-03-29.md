## Durable memory flush — 2026-03-29

- Lucca wants Tobias to be aggressively proactive for research-heavy/uncertain questions: use multiple cheap subagents freely (Haiku, GPT-5.4 mini/OAuth if available), prioritizing official docs plus recent community signals before answering confidently.
- Lucca explicitly wants Tobias to behave as an operational multiplier, not a reactive LLM that requires manual management.
- We built a first MVP of a task watchdog under `task_watchdog/` with:
  - `config.json`
  - `active_tasks.json`
  - `task_history.jsonl`
  - `watchdog.py`
  - `registry_helpers.py`
  - `register_subagent_task.py`
  - `watchdog_launch_audit.py`
  - README documenting spawn -> register -> confirm and launch audit safety layers.
- The watchdog concept is not yet reliable in production. Main failure modes discovered:
  - cron delivery to Telegram errored with `Outbound not configured for channel: telegram`
  - stale/zombie tasks remained in `active_tasks.json` and produced false notifications later
  - some launched tasks were not registered, proving workflow integration is still incomplete
  - a long-run validation script failed due to Python encoding / syntax issues, so the validation test was not valid.
- A stale zombie task named `Claude Code sidecar viability research (retest)` remained marked as running in `active_tasks.json` even though it had completed, causing false Telegram updates on 2026-03-29; Lucca was disturbed by these ghost notifications.
- Lucca wants truly separate conversations/sessions by subject. Research conclusion: Telegram private DM is poor for this; a private Telegram group with Topics is the best route if he wants visible separation inside Telegram.
- For Telegram topics/groups with OpenClaw, the key config findings were:
  - parent group allowlisting covers all topics by default
  - `requireMention` must be `false` if Lucca does not want to mention the bot each time
  - BotFather privacy must be disabled (Lucca says it already is)
  - current config likely still blocks topic replies because `groupPolicy` is `allowlist` and `groups.*.requireMention` had been true; group/topic response remains unresolved.
- Lucca is considering/likes the idea of a dedicated independent Research Agent with preset subagents (different models, skills, search goals) so Tobias can trigger it instead of improvising research ad hoc each time.
- Research on Claude Code sidecar usage concluded: using Claude Code subscription locally as a premium sidecar/worker for OpenClaw is viable for strong long-form tasks (e.g., script writing), but should be treated as a local premium worker with fallback, not as the universal backbone, due to auth/headless/operational fragility.
