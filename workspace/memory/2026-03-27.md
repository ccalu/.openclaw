## Preference learned — task progress updates

Lucca wants explicit progress communication for spawned subagents / longer tasks:
- always notify when a launched task/subagent is finished
- while still running, send an update every 10 minutes
- applies to cases like spawned subagents or long-running background tasks

Operational implication for Tobias:
- do not silently leave background work running without status
- if runtime/tooling cannot support exact timed pushes automatically, state that clearly and emulate as closely as possible when possible
